# 数据库设计文档

## 数据库选型

### 开发阶段：SQLite
- **优点**：无需安装，零配置，轻量级
- **适用场景**：开发、测试、小规模应用

### 生产环境：MySQL
- **优点**：性能更好，支持并发，功能完善
- **适用场景**：生产环境、大规模数据

## 表结构设计

### 1. 用户表 (users)

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL COMMENT '用户名',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希',
    email VARCHAR(100) COMMENT '邮箱',
    real_name VARCHAR(50) COMMENT '真实姓名',
    phone VARCHAR(20) COMMENT '手机号',
    role VARCHAR(20) DEFAULT 'operator' COMMENT '角色：admin/manager/operator',
    status VARCHAR(20) DEFAULT 'active' COMMENT '状态：active/inactive',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '更新时间',
    last_login TIMESTAMP COMMENT '最后登录时间'
);
```

**字段说明**：
- `role`：角色权限
  - `admin`：管理员（所有权限）
  - `manager`：调度员（订单管理、路线规划）
  - `operator`：操作员（查看、录入）

---

### 2. 节点表 (nodes)

```sql
CREATE TABLE nodes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '节点名称',
    code VARCHAR(50) UNIQUE COMMENT '节点编码',
    type VARCHAR(20) NOT NULL COMMENT '类型：warehouse/station/transfer/customer',
    address VARCHAR(255) COMMENT '详细地址',
    province VARCHAR(50) COMMENT '省份',
    city VARCHAR(50) COMMENT '城市',
    district VARCHAR(50) COMMENT '区县',
    longitude DECIMAL(10, 7) COMMENT '经度',
    latitude DECIMAL(10, 7) COMMENT '纬度',
    contact_person VARCHAR(50) COMMENT '联系人',
    contact_phone VARCHAR(20) COMMENT '联系电话',
    capacity INTEGER COMMENT '容量（吨）',
    status VARCHAR(20) DEFAULT 'active' COMMENT '状态：active/inactive',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_by INTEGER COMMENT '创建人',
    FOREIGN KEY (created_by) REFERENCES users(id)
);
```

**节点类型**：
- `warehouse`：仓库
- `station`：配送站
- `transfer`：中转站
- `customer`：客户点

---

### 3. 路线表 (routes)

```sql
CREATE TABLE routes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '路线名称',
    code VARCHAR(50) UNIQUE COMMENT '路线编码',
    start_node_id INTEGER NOT NULL COMMENT '起点节点ID',
    end_node_id INTEGER NOT NULL COMMENT '终点节点ID',
    distance DECIMAL(10, 2) COMMENT '距离（公里）',
    duration DECIMAL(10, 2) COMMENT '预计时长（小时）',
    road_type VARCHAR(20) COMMENT '道路类型：highway/national/provincial/city',
    toll_cost DECIMAL(10, 2) DEFAULT 0 COMMENT '过路费（元）',
    fuel_cost DECIMAL(10, 2) DEFAULT 0 COMMENT '油费（元）',
    total_cost DECIMAL(10, 2) DEFAULT 0 COMMENT '总成本（元）',
    status VARCHAR(20) DEFAULT 'active' COMMENT '状态：active/inactive/construction',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_by INTEGER,
    FOREIGN KEY (start_node_id) REFERENCES nodes(id),
    FOREIGN KEY (end_node_id) REFERENCES nodes(id),
    FOREIGN KEY (created_by) REFERENCES users(id)
);
```

---

### 4. 路段表 (route_segments)

```sql
CREATE TABLE route_segments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    route_id INTEGER NOT NULL COMMENT '所属路线ID',
    sequence INTEGER NOT NULL COMMENT '路段顺序',
    from_node_id INTEGER NOT NULL COMMENT '起点节点ID',
    to_node_id INTEGER NOT NULL COMMENT '终点节点ID',
    distance DECIMAL(10, 2) COMMENT '距离（公里）',
    duration DECIMAL(10, 2) COMMENT '时长（小时）',
    road_name VARCHAR(100) COMMENT '道路名称',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (route_id) REFERENCES routes(id),
    FOREIGN KEY (from_node_id) REFERENCES nodes(id),
    FOREIGN KEY (to_node_id) REFERENCES nodes(id)
);
```

**说明**：一条路线由多个路段组成，用于详细记录路径信息。

### 5. 车辆表 (vehicles)

```sql
CREATE TABLE vehicles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    plate_number VARCHAR(20) UNIQUE NOT NULL COMMENT '车牌号',
    vehicle_type VARCHAR(50) COMMENT '车辆类型',
    brand VARCHAR(50) COMMENT '品牌',
    model VARCHAR(50) COMMENT '型号',
    capacity_weight DECIMAL(10, 2) COMMENT '载重（吨）',
    capacity_volume DECIMAL(10, 2) COMMENT '容积（立方米）',
    driver_name VARCHAR(50) COMMENT '司机姓名',
    driver_phone VARCHAR(20) COMMENT '司机电话',
    status VARCHAR(20) DEFAULT 'available' COMMENT '状态：available/in_use/maintenance',
    current_longitude DECIMAL(10, 7) COMMENT '当前经度',
    current_latitude DECIMAL(10, 7) COMMENT '当前纬度',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6. 订单表 (orders)

```sql
CREATE TABLE orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_number VARCHAR(50) UNIQUE NOT NULL COMMENT '订单编号',
    customer_name VARCHAR(100) COMMENT '客户名称',
    customer_phone VARCHAR(20) COMMENT '客户电话',
    pickup_node_id INTEGER NOT NULL COMMENT '取货节点ID',
    delivery_node_id INTEGER NOT NULL COMMENT '送货节点ID',
    cargo_name VARCHAR(100) COMMENT '货物名称',
    cargo_type VARCHAR(50) COMMENT '货物类型',
    weight DECIMAL(10, 2) COMMENT '重量（吨）',
    volume DECIMAL(10, 2) COMMENT '体积（立方米）',
    priority VARCHAR(20) DEFAULT 'normal' COMMENT '优先级：normal/urgent/express',
    status VARCHAR(20) DEFAULT 'pending' COMMENT '状态：pending/assigned/in_transit/delivered/cancelled',
    recommended_route_id INTEGER COMMENT '推荐路线ID',
    actual_route_id INTEGER COMMENT '实际路线ID',
    vehicle_id INTEGER COMMENT '分配车辆ID',
    estimated_cost DECIMAL(10, 2) COMMENT '预估成本',
    actual_cost DECIMAL(10, 2) COMMENT '实际成本',
    pickup_time TIMESTAMP COMMENT '取货时间',
    delivery_time TIMESTAMP COMMENT '送达时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_by INTEGER,
    FOREIGN KEY (pickup_node_id) REFERENCES nodes(id),
    FOREIGN KEY (delivery_node_id) REFERENCES nodes(id),
    FOREIGN KEY (recommended_route_id) REFERENCES routes(id),
    FOREIGN KEY (actual_route_id) REFERENCES routes(id),
    FOREIGN KEY (vehicle_id) REFERENCES vehicles(id),
    FOREIGN KEY (created_by) REFERENCES users(id)
);
```

**订单状态流转**：
```
pending（待分配）→ assigned（已分配）→ in_transit（运输中）→ delivered（已送达）
                                    ↓
                                cancelled（已取消）
```

### 7. 路线推荐记录表 (route_recommendations)

```sql
CREATE TABLE route_recommendations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_id INTEGER NOT NULL COMMENT '订单ID',
    algorithm VARCHAR(50) COMMENT '使用的算法',
    recommended_routes TEXT COMMENT '推荐路线列表（JSON格式）',
    calculation_time_ms INTEGER COMMENT '计算耗时（毫秒）',
    factors TEXT COMMENT '考虑因素（JSON格式）',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES orders(id)
);
```

---

### 8. 操作日志表 (operation_logs)

```sql
CREATE TABLE operation_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER COMMENT '操作用户ID',
    action VARCHAR(50) NOT NULL COMMENT '操作类型',
    target_type VARCHAR(50) COMMENT '目标类型',
    target_id INTEGER COMMENT '目标ID',
    details TEXT COMMENT '详细信息（JSON格式）',
    ip_address VARCHAR(50) COMMENT 'IP地址',
    user_agent VARCHAR(255) COMMENT '用户代理',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

**操作类型**：
- `login`：登录
- `logout`：登出
- `create`：创建
- `update`：更新
- `delete`：删除
- `view`：查看

## 索引设计

```sql
-- 用户名索引
CREATE INDEX idx_users_username ON users(username);

-- 节点位置索引（用于地图查询）
CREATE INDEX idx_nodes_location ON nodes(longitude, latitude);

-- 节点类型索引
CREATE INDEX idx_nodes_type ON nodes(type);

-- 路线起点终点索引
CREATE INDEX idx_routes_nodes ON routes(start_node_id, end_node_id);

-- 订单状态索引
CREATE INDEX idx_orders_status ON orders(status);

-- 订单创建时间索引
CREATE INDEX idx_orders_created_at ON orders(created_at);

-- 操作日志用户索引
CREATE INDEX idx_logs_user ON operation_logs(user_id);

-- 操作日志时间索引
CREATE INDEX idx_logs_created_at ON operation_logs(created_at);
```

## 数据库迁移

### SQLite → MySQL 迁移步骤

1. 导出SQLite数据
```bash
sqlite3 logistics.db .dump > dump.sql
```

2. 转换SQL语法
   - `AUTOINCREMENT` → `AUTO_INCREMENT`
   - `TEXT` → `VARCHAR` 或 `TEXT`
   - 日期函数调整

3. 导入MySQL
```bash
mysql -u root -p logistics < dump.sql
```

## 数据规模估算

### 小规模（SQLite适用）
- 用户：< 50
- 节点：< 1,000
- 路线：< 5,000
- 订单：< 10,000/年

### 中规模（MySQL推荐）
- 用户：50 - 500
- 节点：1,000 - 10,000
- 路线：5,000 - 50,000
- 订单：10,000 - 100,000/年
