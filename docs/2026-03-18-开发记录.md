# 2026-03-18 成本分析报表调试记录

## 问题现象
前端访问成本分析页面时，所有 API 返回 500 错误：
- `/api/cost/overview`
- `/api/cost/trend`
- `/api/cost/distribution`
- `/api/cost/by-node`
- `/api/cost/optimization-suggestions`

## 根本原因
1. **数据库文件为空**：`logistics.db` 是 0 字节空文件
2. **Route 模型字段名不匹配**：模型定义 `estimated_time`，数据库表是 `duration`
3. **Order 模型字段不匹配**：模型定义了 `estimated_cost`、`actual_cost`、`recommended_route_id` 等字段，但数据库表没有这些字段

## 已完成修复 ✅

### 1. 修复 Route 模型
- 将 `estimated_time` 改为 `duration`，匹配数据库表结构

### 2. 修复 Order 模型
- 移除不存在的字段：`estimated_cost`、`actual_cost`、`recommended_route_id`、`actual_route_id`、`pickup_time`、`delivery_time`、`created_by`
- 使用数据库实际字段：`estimated_delivery`、`actual_delivery`、`notes`

### 3. 修复 init_db.py
- 更新 Route 创建代码，使用 `duration` 字段
- 更新 Order 创建代码，移除不存在的字段

### 4. 重写成本分析路由
- `cost.py` 完全重写，基于现有数据库结构
- 新增 `calculate_route_cost()` 函数计算路线成本
- 新增 `estimate_order_cost()` 函数估算订单成本
- 所有 API 现在可以正常工作

### 5. 重新初始化数据库
- 删除空的数据库文件
- 运行 `init_db.py` 创建表和示例数据
- 示例数据：5个节点、4条路线、3辆车、3个订单
